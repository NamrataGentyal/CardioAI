📋 About the Project

About CardioAI

A machine learning web app built to predict heart failure mortality risk from clinical patient data, designed to assist in early medical decision-making.

💡 Project Overview

CardioAI is a Flask-based web application that leverages supervised ML models trained on the Heart Failure Clinical Records dataset. It predicts whether a patient is at risk of a death event within a clinical follow-up period based on 12 clinical features.

The application accepts patient data through a clean UI, validates all inputs, and returns a colour-coded risk result with a probability confidence score.

📊 Dataset

  • 299 patient records collected from hospital follow-ups
  • 12 clinical features per patient
  • Binary classification target: DEATH_EVENT (0 or 1)
  • Features include ejection fraction, serum creatinine, serum sodium, CPK, platelets, age, anaemia, diabetes, hypertension, sex, smoking, and follow-up time

⚙ ML Pipeline

01

Data Loading

CSV loaded with pandas, features extracted and validated.

02

Train / Test Split

80/20 stratified split to preserve class balance.

03

Scaling

StandardScaler normalises all features to zero mean and unit variance.

04

Grid Search

5-fold CV GridSearchCV on Random Forest optimising ROC-AUC.

05

Model Selection

Best model chosen across RF, GB, LR, and SVM by ROC-AUC score.

06

Artifacts

model.pkl, scaler.pkl, and model_stats.json persisted for the web app.

🔨 Tech Stack
